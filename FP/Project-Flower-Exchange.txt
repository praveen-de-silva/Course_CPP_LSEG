SYSTEM OVERVIEW
================================================================

The Flower Exchange is a simplified trading system that:
  1. Reads orders from an input file (orders.csv)
  2. Processes each order against an Order Book
  3. Produces an output execution report file (execution_rep.csv)

Two conceptual components:
  - Trader Application  : Submits buy/sell orders (simulated via orders.csv)
  - Exchange Application: Processes orders and generates execution reports

For this project, the simpler implementation (2nd option) is used:
  orders.csv  →  Exchange Application  →  execution_rep.csv

================================================================
ORDER BOOK MECHANICS
================================================================

- One Order Book per instrument (5 books total: Rose, Lavender,
  Lotus, Tulip, Orchid)
- Each book has a BUY side (blue) and a SELL side (pink)
- Initially empty; orders accumulate over time

SORTING RULES:
  - Buy Side  : Sorted in DESCENDING price order
                (highest buy price = most attractive = top priority)
  - Sell Side : Sorted in ASCENDING price order
                (lowest sell price = most attractive = top priority)
  - Tie-break : Time priority (earlier order gets higher priority)

ORDER TYPES:
  - Passive Order  : Incoming order that cannot match; goes into
                     the Order Book with status "New"
  - Aggressive Order: Incoming order that matches an existing order;
                      executes immediately

EXECUTION STATUS LOGIC:
  - New    : Order entered the book with no match
  - Fill   : Order fully matched and executed
  - PFill  : Order partially matched (remaining qty stays in book)
  - Rejected: Order failed validation checks


================================================================
MATCHING LOGIC (Price-Time Priority)
================================================================

When a new order arrives:
  1. Validate the order (see validations below)
  2. If BUY order → try to match against SELL side (lowest ask first)
     Match condition: buy_price >= sell_price
  3. If SELL order → try to match against BUY side (highest bid first)
     Match condition: sell_price <= buy_price
  4. Execution price is determined by the passive (resting) order's price
  5. If fully matched  → status = Fill  (for both aggressor & passive)
  6. If partially matched → status = PFill for aggressor,
                            Fill for passive (when passive fully consumed)
  7. If no match → status = New, order sits in the book


================================================================
INPUT VALIDATION RULES
================================================================

An order is REJECTED if any of the following are true:
  ✗ Missing any required field
  ✗ Instrument is not one of: Rose, Lavender, Lotus, Tulip, Orchid
  ✗ Side is not 1 (Buy) or 2 (Sell)
  ✗ Price is not > 0.0
  ✗ Quantity is not a multiple of 10
  ✗ Quantity is < 10 or > 1000

A rejected order gets a Rejected Execution Report with a Reason field.
Common reasons: "Invalid instrument", "Invalid side", "Invalid price",
                "Invalid size"

RECOMMENDED FILE/CLASS STRUCTURE:
----------------------------------

  main.cpp
  ├── CSVReader.h / CSVReader.cpp
  │     → Reads orders.csv line by line, parses fields,
  │       returns vector<Order>
  │
  ├── Order.h
  │     → Struct/class with fields: clientOrderId, instrument,
  │       side, price, quantity
  │
  ├── ExecutionReport.h
  │     → Struct/class for output: orderId, clientOrderId,
  │       instrument, side, price, quantity, status, reason,
  │       transactionTime
  │
  ├── OrderBook.h / OrderBook.cpp
  │     → One per instrument
  │     → Two containers: buySide, sellSide
  │     → Use std::multiset or std::deque with custom comparator
  │     → matchOrder(Order&) → returns vector<ExecutionReport>
  │
  ├── Exchange.h / Exchange.cpp
  │     → Holds map<string, OrderBook> for 5 instruments
  │     → validateOrder(Order&) → bool + reason
  │     → processOrder(Order&) → vector<ExecutionReport>
  │     → generateOrderId() → "ord1", "ord2", etc.
  │
  └── CSVWriter.h / CSVWriter.cpp
        → Takes vector<ExecutionReport>, writes execution_rep.csv

DATA STRUCTURES (Recommendations):
------------------------------------
  - Buy Side  : std::multiset with comparator: higher price first,
                then earlier time (use sequence number)
                → e.g. std::set<Order, BuyComparator>
  - Sell Side : std::multiset with comparator: lower price first,
                then earlier time
                → e.g. std::set<Order, SellComparator>

MATCHING ALGORITHM (pseudocode):
----------------------------------
  processOrder(incoming):
    1. validate → if fail, emit Rejected report, return
    2. emit "New" report for incoming order
    3. if incoming.side == BUY:
         while sellSide not empty AND
               sellSide.top().price <= incoming.price AND
               incoming.remainingQty > 0:
             matchQty = min(incoming.remainingQty, sell.qty)
             emit Fill/PFill for incoming (at sell.price)
             emit Fill/PFill for passive sell order
             update quantities; remove filled passive orders
    4. if incoming.side == SELL: (mirror of above with BUY side)
    5. if incoming still has remainingQty > 0:
         add to appropriate side of order book

TRANSACTION TIME:
------------------
  Use <chrono> and <iomanip> to generate timestamp:
    auto now = std::chrono::system_clock::now();
    Format: YYYYMMDD-HHMMSS.sss

ORDER ID GENERATION:
---------------------
  Simple global counter: "ord" + to_string(++counter)
  Use std::atomic<int> if ever multi-threaded

PERFORMANCE TIPS (for large input test):
-----------------------------------------
  - Use reserve() on vectors where size is known
  - Use std::ios::sync_with_stdio(false) and cin.tie(NULL)
  - Avoid unnecessary string copies (use const ref)
  - Use std::multiset for O(log n) insert/lookup instead of
    linear scan over vectors
  - Batch-write execution reports (single file write at end)
  - Avoid repeated file open/close operations

SAMPLE MAIN FLOW:
------------------
  int main() {
    CSVReader reader("orders.csv");
    vector<Order> orders = reader.readAll();

    Exchange exchange;
    vector<ExecutionReport> allReports;

    for (auto& order : orders) {
      auto reports = exchange.processOrder(order);
      allReports.insert(allReports.end(),
                        reports.begin(), reports.end());
    }

    CSVWriter writer("execution_rep.csv");
    writer.writeAll(allReports);
    return 0;
  }
